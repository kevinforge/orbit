## v0.9.3

*当前开发版本，尚未正式发布。包含 v0.9.2 之后的全部改动。*

### 新功能

- **在输入框直接粘贴图片。** 把 PNG / JPEG / WebP 图片拖入或粘贴到输入框，发送前可预览，被指派的数字员工会连同文字一起收到图片。
- **一键复制数字员工。** 新增复制按钮，可把任意数字员工的配置克隆成一份默认关闭的新模板，方便做变体而不必重新填写。
- **真正停掉正在跑的任务。** 打断按钮现在会直接结束正在运行的进程（而不只是清队列），让失控的任务立刻停下、数字员工随即可以接手新工作。

### 日常体验更顺

- **直达数字员工设置。** 鼠标悬停在某个数字员工上，点齿轮图标即可直接打开它的配置。
- **缺工具时提示更清楚。** 当某个 CLI（Claude Code、Codex、CodeBuddy）未安装时，会显示醒目的提示条和安装按钮，不再是小链接。
- **支持 macOS 选目录。** 在 macOS 上也能像 Windows 一样选择工作区目录。
- **长内容更整齐。** 过长的代码块和链接不再撑破消息版面。
- **会话顺序不乱跳。** 点击某个会话不会把它顶到列表最上方。
- **界面中文更统一。** 整个应用（含数字员工设置页）统一为中文，"数字员工" 取代了旧的 "智能体" 措辞。

### 修复与稳定性

- **Codex 回答更干净。** Codex 的内部"旁白"不再混进你看到的答案；数字员工之间转交的消息也能在历史中正确保留。
- **打断反馈更安静。** 停止协作时不再弹出费解的内部提示，按钮直接显示 "已打断" 状态。
- **设置处处同步。** 修改某个数字员工的配置后，工作区内所有会话都会即时生效。
- **终端记录可关闭。** 新增工作区开关，不需要时可以关掉终端日志记录。
- **协调者更聪明。** 别的数字员工提到 `@user:` 时，不再误伤协调者；每个会话只允许一个协调者，内置的 PM 角色也被强化为真正的产品负责人。
- **附件更安全。** 上传的图片会按真实文件头校验（而非只看后缀），每个会话的草稿附件有数量上限，文件处理也不再卡住应用。

## v0.9.2 — 2026-06-04

### 新功能

- **迎来协调者角色。** 可以为工作区配置一个专门的协调（主管）数字员工：当消息没有指派给任何人、或某个数字员工卡住时，它会自动介入，让事情继续推进，省去你时时盯着的麻烦。
- **打断协作链，立刻说新事。** 当多个数字员工来回自动转交时，新的打断按钮会当场叫停后续转交链，你可以马上交代新任务；当前正在跑的任务仍会完成并展示结果。

### 改进

- **数字员工彼此更懂对方。** 交给每个数字员工的内部上下文被重组成清晰的分块结构，协作质量更高、误解更少。
- **历史不再无限膨胀。** 消息和终端记录按时间分片存储，并按时间与数量自动淘汰；旧消息通过"加载更早的消息"按钮按需加载，长会话也能保持轻快、有界。

## v0.9.1 — 2026-06-02

### 新功能

- **每个工作区一套指令与规则。** 可为每个工作区单独设置系统提示词和规则，数字员工自动套用，省去每次重复说明。
- **取消任务。** 新增取消按钮，可去掉某个数字员工队列里等待的任务（或停止正在运行的），队列会正确地接着跑下一个。

### 改进与修复

- **Codex 答案更稳。** 即使 Codex 的输出格式很刁钻，最终答案也能被稳定提取；紧凑版面也更清爽了。
- **权限设置不再丢。** 数字员工的权限配置现在能正确保存；当所需 CLI 未安装时，Orbit 会清楚标出该数字员工"不可用"。

## v0.9.0 — 2026-06-01

**Orbit 首个公开发布版本** —— 一个本地优先的协作控制台，在一个共享会话里
协调多个 CLI 后端的数字员工（Claude Code、Codex、CodeBuddy）。

- **一个频道，多个数字员工。** 用 `@名称:`（带冒号）来派活（冒号表示"去做这件事"），用不带冒号的 `@名称` 仅提及，或用 `@all:` 一次性派给所有人。
- **四个内置数字员工。** `@pm:`（产品）、`@architect:`（架构）、`@developer:`（开发）、`@tester:`（测试）开箱即用，各自权限不同——只有 developer 能写文件、提交代码、安装依赖。
- **自定义数字员工。** 可在界面里创建并配置自己的数字员工，包括权限。
- **工作区与会话。** 把工作划分到相互隔离的工作区和会话中，各自保留独立历史。
- **有序的任务队列。** 每个数字员工一次只跑一个任务，其余自动排队。
- **接着上次继续。** 会话跨次保留，数字员工可以续接之前的工作。
- **回答更易读。** 数字员工的回复以 Markdown 渲染，带语法高亮代码块，工具活动过程也实时可见。
